# OutfitScan
I wanted to create an app that uses Ximilar, RapidAPI, and googles API to scan images of articles of clothing and show pinterest style outfits 
with similar clothing items to give outfit ideas that you can incorperate the article of clothing into.  
OutfitScan is an iOS application designed to analyze images of clothing and generate outfit inspiration based on the detected features. The purpose of the app is to simplify the process of finding styling ideas by allowing users to upload a photo and receive visually similar outfit suggestions. By combining image recognition, color analysis, and search functionality, the app creates a seamless experience that bridges computer vision and fashion discovery.
The application begins by allowing the user to select an image from their photo library. Once a photo is chosen, it is displayed on the main screen as a preview. The user can then initiate the analysis process by pressing the “Find Outfit” button. At this stage, the app sends the image to multiple APIs simultaneously in order to extract meaningful information from the photo.
The first component of the analysis involves identifying clothing items within the image. This is accomplished through a fashion recognition API provided through RapidAPI. The API processes the uploaded image and returns labels representing detected items, along with confidence scores. These labels are then filtered and organized to determine the most relevant clothing item in the image.
At the same time, the app performs color analysis using the Ximilar dominant color API. This step extracts the most prominent colors present in the clothing and assigns approximate percentages to each. The result is a simplified summary of the image’s color composition, which is later used to enhance search accuracy and provide more descriptive output.
After both the clothing and color data have been processed, the app combines this information into a concise, user-friendly description. For example, if the system detects a top and identifies gray as the dominant color, it will generate a label such as “Grey Top.” This description serves as the foundation for the next step of the application’s functionality.
Using the generated description, the app constructs a search query and sends it to an external search service powered by SearchAPI. This service retrieves image-based results from Google Images that match the query. The goal is to provide outfit inspiration that is visually and stylistically similar to the original uploaded image.
The results are then displayed in a secondary view within the app. This interface presents a grid of outfit images along with brief titles. Each image is interactive, allowing the user to tap on it and open the original source link in a web browser. This design enables users to easily explore styling ideas and access additional content beyond the app itself.
In addition to its core functionality, OutfitScan includes error handling and fallback messaging to ensure a smooth user experience. If an API fails to return data or if certain features are not configured, the app provides clear feedback rather than leaving the user without guidance. The interface is designed to remain clean and readable, prioritizing the final results rather than exposing raw API data.
From a development perspective, running OutfitScan requires a properly configured iOS development environment. The project should be opened in Xcode on a Mac with a valid iOS simulator or a connected physical device. Before running the app, a developer must obtain API keys for the services used in the application, including RapidAPI for clothing detection, Ximilar for color analysis, and SearchAPI for retrieving outfit inspiration results. These keys must be inserted into the corresponding constants in the ContentView file where the network requests are configured.
Once the API keys have been added, the developer can build and run the project directly through Xcode. The app will launch in the simulator or on the device, allowing full interaction with the image picker and API-driven features. It is important to note that network connectivity is required for the app to function properly, as all analysis and search operations depend on external API calls. If any API key is missing or invalid, the app will still run but will display error messages or incomplete results.
Overall, OutfitScan demonstrates how multiple APIs and technologies can be integrated into a single application to create a practical and engaging user experience. By combining image recognition, color detection, and search capabilities, the app transforms a simple photo into a source of fashion inspiration while also serving as a functional example of modern iOS development.
